01 Figma MCP Setup
Remote MCP recommended
Desktop MCP
Figma Console MCP
PrerequisitesBefore you start
Requires a Figma account with a Dev, Professional, Org, or Enterprise seat. Supported clients: Claude Code CLI or Cursor.
Free Starter / Viewer seats are limited to 6 remote MCP tool calls per month. Upgrade to at least a Professional Dev seat for real workflow use.
Add remote server in Claude CodeStep 1
Run in terminal (Claude Code CLI must already be installed):
claude mcp add --transport http figma-remote-mcp https://mcp.figma.com/mcp
Restart Claude Code. Run /mcp — the figma-remote-mcp server shows as disconnected until authenticated.
Authenticate with FigmaStep 2
Inside Claude Code, run /mcp, select the Figma server, choose Authenticate. Browser opens — log in to Figma and click Allow access. Run /mcp again to confirm status shows connected.
Add remote server in CursorStep 3 — if using Cursor
Option A — plugin (easiest): In Cursor's agent chat, type @figma. Cursor prompts you to install the Figma plugin which bundles MCP + Agent Skills.

Option B — manual: Cursor Settings → MCP tab → Add new global MCP server:
{ "mcpServers": { "figma": { "url": "https://mcp.figma.com/mcp" } } }
The plugin option also installs Agent Skills — guidance for implementing designs, Code Connect, and design system rules. Prefer it over manual config.
VerifyConfirm it works
Copy a Figma frame link (right-click any frame → Copy link). In Claude Code or Cursor, prompt:
Implement the component at this Figma URL as a React component: https://figma.com/design/ABC123/...?node-id=X-Y
Enable desktop MCP in FigmaStep 1
Install / update the Figma Desktop app (web version does not support this). Open any Design file, then:
Figma menu (top-left) → Preferences → Enable Dev Mode MCP Server
A toast confirms the server is running at http://127.0.0.1:3845/mcp. The server only runs while Figma Desktop is open.
Connect Claude CodeStep 2
claude mcp add --transport http figma-desktop http://127.0.0.1:3845/mcp
Restart Claude Code, run /mcp to confirm connection.
Connect CursorStep 3 — if using Cursor
Cursor Settings → MCP tab → Add Custom MCP:
{ "mcpServers": { "figma-desktop": { "url": "http://127.0.0.1:3845/mcp" } } }
Desktop MCP is selection-based — whatever frame or layer you have selected in Figma is what the agent sees. Best for real-time, component-by-component work. Remote MCP uses links and is better for async / handoff workflows.
What is Figma Console MCP?Context
A community-built MCP server providing write-access tools — create nodes, update properties, run plugin code, read console logs. Complements the official Figma MCP (which is read-optimised). Used in the XCentium workflow during the Design phase for AI-assisted Figma authoring.
Install Figma Console MCPStep 1
Add to ~/.claude/settings.json:
{ "mcpServers": { "figma-console-mcp": { "command": "npx", "args": ["figma-console-mcp"] } } }
Or via terminal: claude mcp add figma-console-mcp -- npx figma-console-mcp
Configure Figma access tokenStep 2
Figma → Settings → Security → Personal Access Tokens → Generate (File content: Read+Write, Code content: Read).
export FIGMA_ACCESS_TOKEN="figd_your_token_here"
Add to ~/.zshrc or ~/.bashrc to persist.

What are Cursor Rules?Context
Rules (.cursor/rules/*.mdc) are markdown files with YAML front matter — Cursor reads them as persistent agent instructions. Git-committed and shared with the team. Scoped by file glob. Use them to encode coding standards, component patterns, and platform conventions.
CLAUDE.md vs .cursor/rulesKey distinction
CLAUDE.mdRead by Claude Code (CLI + Desktop). Project context, active skills, platform constraints, what to never do. One file per project root.
.cursor/rules/*.mdcRead by Cursor agent. Coding standards, file glob scopes, always-apply flags. Multiple files, each scoped to a pattern.
Both files are git-committed. CLAUDE.md is your planning and spec context. .cursor/rules are your implementation guardrails. Platform-specific templates for both are in Section 04.
